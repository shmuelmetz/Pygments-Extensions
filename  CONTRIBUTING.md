# Contributing to pygments-extensions

Thank you for your interest in contributing to **pygments-extensions**.

This project provides additional lexers, styles, and helpers for use with
[Pygments](https://pygments.org/). Contributions that improve correctness,
coverage, or usability are welcome.

## Development workflow

1. Fork the repository on GitHub.
2. Create a feature branch for the change.
3. Add or update tests where appropriate.
4. Run the test suite locally.
5. Submit a pull request with a clear description of the change.

## Coding style

- Follow the existing style in the surrounding code.
- Prefer explicit, readable code over clever one-liners.
- Keep changes focused and self-contained.

## Commit messages

- Use concise, descriptive commit messages.
- Reference issues where relevant (for example, `Fix #12`).

## Reporting issues

If a bug is found or a feature is desired, open an issue with:

- A clear description of the problem or request.
- Steps to reproduce (for bugs).
- Example code or input where helpful.

Thank you for helping improve pygments-extensions.
